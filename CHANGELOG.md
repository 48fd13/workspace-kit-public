# Changelog

## 0.1.0

- Formalized this repository as a reusable OpenCode/Claude folder-workflow kit.
- Added `_workspace/` as the target project/vault control folder layout.
- Added work and personal workflows, runbooks, context files, and staged output folders.
- Added a general engineering pack for backend, cloud, DevOps, IaC, CI/CD, config, containers, Kubernetes manifest review, observability instrumentation, security-aware review, performance, queues, and resilience.
- Added provider-neutral cloud/DevOps workflows for infrastructure changes, environment provisioning, IAM, DNS/TLS/networking, deployment strategy, and managed service integrations.
- Added lightweight day-to-day development workflows for feature implementation, bug fixes, refactors, and test additions.
- Added reusable skills for skill creation, clear writing, and humanization.
- Added OpenCode global setup guidance and single-agent workflow execution.
- Added templates and scripts for initializing and verifying workspaces.
- Moved kit-level explanation and artifact templates toward `docs/` and `templates/artifacts/` so `~/workspace` can remain the daily router.
- Added strict MWP-style `pipeline` template and stage contract/review artifacts.
