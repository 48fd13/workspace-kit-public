# Code / Config Change Workflow

Use for non-trivial local code, config, docs, test, or workflow changes.

1. Identify outcome and scope.
2. Inspect only relevant files.
3. State assumptions, risks, and intended files/areas.
4. Ask before proceeding when gated.
5. Implement the smallest change that satisfies the request.
6. Validate the touched scope.
7. Summarize changes, validation, deviations, and follow-ups.
