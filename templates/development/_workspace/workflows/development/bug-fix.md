# Bug Fix Workflow

Use for normal bugs with a known symptom, failing test, error, or reproduction.

1. Capture observed behavior, expected behavior, and failure output.
2. Locate the smallest relevant code path.
3. Form one concrete hypothesis at a time.
4. Prefer a failing test that reproduces the bug.
5. Make the smallest root-cause fix.
6. Run focused validation, then broader validation if needed.
7. Summarize root cause, fix, and validation.
