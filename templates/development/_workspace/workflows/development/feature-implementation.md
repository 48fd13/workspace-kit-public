# Feature Implementation Workflow

Use for normal feature work where desired behavior is reasonably clear.

1. Restate the feature.
2. Identify the smallest useful behavior slice.
3. Inspect relevant files and patterns.
4. Add or update tests when useful.
5. Implement the feature.
6. Run the smallest relevant validation set.
7. Summarize behavior changed, files changed, validation, and follow-ups.

Escalate to design/engineering review if the feature changes service boundaries, API contracts, auth, data, deployment, or infrastructure.
