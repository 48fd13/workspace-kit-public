# Refactor Workflow

Use for behavior-preserving cleanup, simplification, renaming, extraction, or structure changes.

1. State the intended refactor and unchanged behavior.
2. Identify tests or validation that protect behavior.
3. Keep the refactor scoped and reversible.
4. Do not mix refactor with feature changes.
5. Make small incremental changes.
6. Run validation.
7. Summarize structural changes and behavior protection.
