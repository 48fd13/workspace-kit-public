# Pipelines

Add pipelines here as named subfolders, one per pipeline. See `docs/authoring/authoring-pipelines.md` in the kit root for what a pipeline is, the folder structure, and the run lifecycle.

- `pipeline-template/` — blank scaffold. Copy it, rename it, and customize the stage contracts to create a new pipeline.
- `workspace-bootstrap/` — a working example, not a scaffold: it researches an unfamiliar target and drafts this workspace's own map, routing table, workflows, and reference context. Read it to see the conventions applied, or run it against a real target.
