# Config

Stable rules for how this pipeline should bootstrap any workspace, regardless of target.

Examples:

- map-conventions.md — house style for workspace-map.md and routing-table.md rows
- workflow-conventions.md — house style for short workflow checklists (see `minor-change.md` for the shape)

These files are Layer 3 reference material. Keep target-specific facts out of here — those belong in the active run's `input/` and stage output, or later in `_workspace/context/project/`.
