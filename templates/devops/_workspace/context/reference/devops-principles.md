# Cloud DevOps Principles

- Map before changing.
- Prefer read-only discovery before action.
- Separate facts, assumptions, risks, and open questions.
- Identify blast radius, ownership, environment, dependencies, and rollback/fix-forward path early.
- Validate with evidence, not confidence.
- Make operational knowledge durable through maps, runbooks, change packets, and incident notes.
- Keep app, platform, infrastructure, CI/CD, security, and observability concerns connected but not mixed.
