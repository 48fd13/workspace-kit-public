# Observability Principles

Observability should help humans make decisions.

Review:

- logs: useful events, correlation IDs, error context
- metrics: saturation, traffic, errors, latency, resource usage
- traces: request flow and dependency latency
- alerts: actionable, owned, linked to runbooks
- dashboards: answer operational questions, not just display data

Distinguish symptoms from causes. Do not infer cause from one signal alone.
